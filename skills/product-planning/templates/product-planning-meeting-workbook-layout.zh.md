# Product Planning Meeting Workbook Template V1

本模板用于 `/product-planning "{ASIN1;ASIN2;ASIN3}" {SITE}` 输出产品立项会议版 Excel。产品方向由三个已验证种子 ASIN 的共同类目、前端卖点、场景和关键词交集推导。

## 版本确认

- 标准版本：`Product Planning Standard Template V1`
- 确认日期：`2026-07-06`
- 首选工作簿：`templates/product_planning_standard_template_V1_beautified.xlsx`
- 兼容备份：`templates/product_planning_standard_template_V1.xlsx`
- 适用命令：`/product-planning "{ASIN1;ASIN2;ASIN3}" {SITE}`
- 标准范围：保留 `意向产品`、`市场分析`、`竞品分析与优化策略`、`SWOT分析`、`ABA排名【季度】`、`MCP原始数据`、`数据来源`；默认删除 `成本试算` 和 `销售计划`。

## 模板文件

- `templates/product_planning_standard_template_V1_beautified.xlsx`（默认）
- `templates/product_planning_standard_template_V1.xlsx`（仅在首选模板不可用时回退）

## Sheet 顺序

1. `意向产品`
2. `市场分析`
3. `竞品分析与优化策略`
4. `SWOT分析`
5. `ABA排名【季度】`
6. `MCP原始数据`
7. `数据来源`

## 第一 Sheet：意向产品

第一 Sheet 按“一个意向产品一行”的方式横向拆解，结构参考用户提供的产品企划表。

### 版式

- 第 1 行：`产品意向｜{DERIVED_PRODUCT_DIRECTION}`
- 第 2 行：数据提炼来源说明
- 第 3 行：`方案 A｜{DERIVED_PRODUCT_DIRECTION}`
- 第 4 行：表头
- 第 5 行：意向产品拆解内容
- 第 7 行：填写口径说明

表头样式固定：功能一至功能四为已验证卖点，使用灰底红字；功能五至功能七为未满足需求，使用亮黄底红字并保留“未满足需求”标注。

### 字段

| 字段 | 输出来源 | 填写规则 |
|---|---|---|
| 参考图片 | 第一个种子 ASIN 的 Amazon 前端首图 | 默认以命令中的第一个 ASIN 为视觉基准；前端页面和 ASIN 校验一致后插入首图，并在 `数据来源` 登记为前端快照。抓取失败时保留 `N/A`，不得换用无关图片 |
| 价格定位 | Amazon 前端价格 + MCP 竞品价格带 | 只写市场定位，不写利润 |
| 客户人群 | Listing 场景 + MCP 用户评论 | 从使用场景、评论身份、购买目的提炼 |
| 功能一 | Amazon 前端产品介绍/图片卖点 + 高星评论 | 已被产品明示或用户认可的功能 |
| 功能二 | Amazon 前端产品介绍/图片卖点 + 高星评论 | 已被产品明示或用户认可的功能 |
| 功能三 | Amazon 前端产品介绍/图片卖点 + 高星评论 | 已被产品明示或用户认可的功能 |
| 功能四 | Amazon 前端产品介绍/图片卖点 + 高星评论 | 已被产品明示或用户认可的功能 |
| 功能五（未满足需求） | MCP 低星评论 | 可转化为产品改进的用户痛点 |
| 功能六（未满足需求） | MCP 低星评论 | 可转化为产品改进的用户痛点 |
| 功能七（未满足需求） | MCP 低星评论 | 可转化为产品改进的用户痛点 |

### 证据优先级

1. Amazon 前端：标题、五点、A+、图片卖点、价格、变体、适用场景。
2. SellerSprite MCP 评论：1-3 星差评用于未满足需求，4-5 星好评用于已验证卖点。
3. SellerSprite MCP 竞品与关键词：只用于辅助价格、人群和需求判断。

禁止把没有来源的功能点、成本、利润、首批数量写入第一 Sheet。

## 第二 Sheet：市场分析

第二 Sheet 用于判断“该具体类目是否值得进入”，必须在先确定 `nodeIdPath` 后执行。页面严格按照 7 个维度组织，每个维度都包含：

### 类目选择规则

1. 从 3 个用户种子和 1 个系统补充竞品的 `asin_detail.nodeIdPath` 建立候选路径表。
2. 优先选择重复出现、与产品形态直接一致的最深叶子节点；重复叶子节点相同时，再比较标题相关性与目标功能覆盖。
3. 没有重复叶子节点时，才选择覆盖多数用户种子的最小共同父级。
4. 共同父级若包含大量无关子类，或均价/商品结构明显被非目标商品扭曲，则判定为“范围过宽”，不得直接用于市场统计；改用最相关叶子节点，并把其他种子类目列为相邻对照。
5. 市场页顶部必须写明最终节点、覆盖率、相邻类目及排除其他候选节点的原因。

1. MCP 接口/命令来源。
2. 数据表。
3. 图表。
4. 表格下方的“产品经理点评”。

### 固定分析维度

| 顺序 | 模块 | 必调/优先 MCP 接口 | 输出内容 | 产品经理点评方向 |
|---|---|---|---|---|
| 1 | 销售趋势 | `market_product_demand_trend`, `market_research_statistics` | 月份、销量、销售额、平均BSR/排名 | 判断销量是否稳定、是否季节性集中、是否适合当前时间进入 |
| 2 | 关键词 | `keyword_miner`, `keyword_research_trends`, `aba_research_trend` | 关键词、搜索量、购买量、PPC、商品数 | 判断需求是否真实、关键词是否增长、CPC是否可承受 |
| 3 | 上架时间分布 | `market_listing_date_distribution` | 上架时长、产品数量、销量占比、平均销量占比 | 判断老品垄断、新品窗口、生命周期长度 |
| 4 | 上架趋势分布 | `market_listing_trend_distribution` | 年份、产品数量、销量占比 | 判断近期新品进入速度、是否出现供给加速 |
| 5 | 评分数分析 | `market_ratings_count_distribution` | 评分数区间、产品数量、销量占比、平均销量占比 | 判断评论壁垒和新品突破难度 |
| 6 | 评分值分布 | `market_rating_distribution` | 评分值区间、产品数量、销量占比、平均销量占比 | 判断产品质量成熟度和差评突破机会 |
| 7 | 价格区间 | `market_price_distribution`, `competitor_lookup` | 价格区间、产品数量、销量占比、销售额占比 | 判断主力价格带、利润空间和差异化定价位置 |

### 填写规则

- 所有图表必须由对应表格数据生成，不允许只贴截图。
- 每个维度下方必须写 1-3 条产品经理点评，格式为 `1、...`。
- 点评必须转成产品决策语言，例如：进入/观察、主攻价格带、避开高壁垒区间、先做小样验证、需要供应链补数。
- MCP 返回空字段时写 `N/A`，并在 `数据来源` Sheet 标注缺口和对结论的影响。
- 如果某个 MCP 接口不可用，只能在该维度标注“接口不可用/数据缺口”，不得用猜测数据补图。
- 市场分析数据只能来自最终选择的类目节点：优先使用 `Camera Mounts & Clamps` 叶子节点，也可在叶子数据不足时使用其直接父节点 `Tripod & Monopod Accessories`。`market_product_demand_trend` 某月份为空时应先调整月份参数重试；销售表通过同一节点连续 12 个月的 `market_research_statistics` 填入 `avgUnits`、`avgRevenue`、`avgBsr`，不得用 ASIN 聚合替代类目数据。
- 关键词区域必须写入 `keyword_miner.keywordList` 的真实返回值；模板令牌、空关键词和全 0 占位行必须在交付前清除，未返回词写 `N/A（精确查询未返回）`。

## 第三 Sheet：竞品分析与优化策略

第三 Sheet 用于把“竞品是谁、为什么能卖、用户哪里不满意、我们怎么改”串成产品动作。命令中的三个分号分隔 ASIN 就是用户指定的代表竞品；不再在流程中重复索要 ASIN。

用户提供的 3 个 ASIN 必须经 `asin_detail` 验证产品存在、站点一致、标题相关和类目节点相关；重复、无效、跨站点或明显无关时要求用户替换。随后使用 `competitor_lookup`，必要时使用 `product_research` 建立候选池，自动补充 1 个必要竞品，最终固定输出 4 个样本。补充样本优先覆盖用户 3 款尚未覆盖的头部评价壁垒、低价冲击或高配功能差异化；找不到合格样本时第 4 行写 `N/A` 并登记数据缺口。

### 版式

1. 顶部：三个用户种子的基础参数对比，包含 ASIN、品牌、上架日、类目路径、价格、评分、评分数/评论数、LQS、Amazon Choice、EBC/视频、变体、BSR、重量和月销量快照。
2. 核心差异化信号：按品牌信号、专业接口、设备兼容、场景定位、材质/便携、融合方向启示横向对比三款种子；未调用的字段统一写 `N/A`。
3. 竞品纵向卡片：固定输出 4 张卡片，前三张保留用户指定 ASIN，第四张标记“系统补充样本”或 `N/A`。每张卡片包含基础快照、用户画像、核心场景、前端图片/截图信息、Listing 卖点、4-5 星好评主题、1-3 星痛点、对标方向和证据 ID。
4. 图片口径：Amazon 前端图片和截图可辅助识别型号、场景、价格、月销或评分，但必须标记为“截图快照”；实时字段以本次 MCP 返回为准。
5. 产品经理结论：3 条以内，必须落到样品验证、Listing 表达、广告/定价动作。

### MCP 数据映射

| 内容 | 必调/优先 MCP 接口 | 填写规则 |
|---|---|---|
| 用户指定竞品 | 前端对话输入 3 个 ASIN + `asin_detail` | 用户 ASIN 不按评分或销量替换；无效、不相关或站点不一致时要求重新输入 |
| 竞品池/系统补充 | `competitor_lookup`, `product_research`, `asin_detail` | 只收录标题和类目节点高度相关的商品；第 4 个样本优先补足评价壁垒、低价或高配维度 |
| 竞品详情 | `asin_detail` | 用于标题、品牌、价格、评分、评分数、LQS、前端卖点 |
| 正向卖点 | `asin_detail.features`, `review` 4-5 星 | 只写前端明示或高星评论确认过的卖点 |
| 负向痛点 | `review` 1-3 星 | 只写低星评论中可转为产品需求的痛点 |
| 优化动作 | 由竞品+评论证据推导 | 必须写成可执行动作，例如结构加强、防刮、避让接口、说明边界 |

### 结论规则

- 最多 3 条，分别覆盖样品验证、Listing 表达、定价或投放中的可用动作。
- 每条都必须能回查到上述 ASIN 的 `asin_detail`、Amazon 前端卖点或 `review` 证据；没有证据时写 `N/A`，不自动生成具体结论。

## 第四 Sheet：SWOT分析

SWOT 不是泛泛写优劣势，而是对前面 Sheet 的证据做决策归纳。每一条必须能追溯到关键词、市场分布、竞品、评论、趋势或明确的数据缺口。

### 版式

1. 2x2 SWOT 矩阵：
   - S：已被 MCP/前端/VOC 支撑的优势。
   - W：当前数据缺口、供应链验证缺口、产品短板。
   - O：关键词机会、价格带机会、评论痛点机会。
   - T：评论壁垒、品牌/知识产权、流量集中度、竞品低价威胁。
2. 决策门槛表：
   - 需求
   - 竞争
   - 差异化
   - 数据完整性
   - 进入结论
3. 每个门槛必须包含：状态、证据、下一步动作。

### MCP 数据映射

| SWOT 模块 | 证据来源 |
|---|---|
| S 优势 | `keyword_miner` 购买率/搜索量、`market_price_distribution` 主力价格带、`review` 好评、`asin_detail.features` |
| W 劣势 | `dataGaps`、MCP 空字段、样品/供应链未验证项 |
| O 机会 | `review` 低星痛点、`market_listing_trend_distribution` 新品供给变化、`market_rating_distribution` 质量缺口 |
| T 威胁 | `market_ratings_count_distribution` 评论壁垒、`monopolyClickRate` 点击集中度、`trademark_*` 合规风险 |

## 第五 Sheet：ABA排名【季度】

第五 Sheet 用于把关键词从“能不能做”拆到“先打哪些词、怎么打、哪些词先观察”。它不是简单关键词列表，而是关键词分层和投放动作页。

### 版式

1. 关键词主表：关键词、搜索量、购买量、购买率、均价、商品数、PPC、供需比、点击集中度、SPR、角色、处理建议。
2. 图表一：关键词搜索量/购买量，判断量级。
3. 图表二：供需比/点击集中度，判断竞争与垄断。
4. ABA 趋势表：按月或按季度放核心词趋势。
5. ABA 趋势图：用于判断需求是否稳定、增长或回落。
6. 产品经理点评：宽词/长尾词/精准广告/否词/待补数据。

### MCP 数据映射

| 内容 | 必调/优先 MCP 接口 | 填写规则 |
|---|---|---|
| 关键词主表 | `keyword_miner` | 必须保留搜索量、购买量、购买率、PPC、商品数、供需比、点击集中度、SPR |
| 趋势 | `aba_research_trend`, `keyword_research_trends` | 趋势字段为空时写 N/A，不画假趋势 |
| 竞品反查词 | `keyword_order`, `traffic_keyword`, `traffic_source` | 用于确认竞品真实转化词；接口失败则写入数据缺口 |
| 处理建议 | 由关键词指标推导 | 宽词、核心长尾、精准词、观察词要分层，不要混成同一类 |

## 第六/第七 Sheet：MCP原始数据 与 数据来源

默认会议版仍保留这两张审计页，位置在 `ABA排名【季度】` 之后。

- `MCP原始数据`：只保存脱敏后的 compact 字段，不保存 API key、token、secret-key URL、本地路径或完整隐私 raw。
- `数据来源`：记录每个 Sheet 用到的 MCP 工具、入参摘要、可用字段、缺口和处理方式。
- 如果某个接口失败，例如凭证、网络、字段为空，必须记录失败原因，不允许用经验值补齐。

## 默认不包含

- `成本试算`
- `销售计划`

成本、销售、头程、FBA、退货率、利润率等内容不属于 SellerSprite MCP 直接输出。除非用户明确要求做财务测算，否则会议版 Excel 不单独生成这两个 Sheet。若用户要求补充，必须使用用户提供的成本/供应链假设，并在 `数据来源` Sheet 标注为人工假设。

## 填充规则

- 所有市场、关键词、竞品、评论、类目结论必须来自 SellerSprite MCP、Amazon 前端、已有源报告或用户提供假设。
- MCP 返回空字段时写 `N/A`，并在 `数据来源` Sheet 标注数据缺口。
- 不允许把旧模板中的样例值当作新报告数据。
- 不允许在 Excel、Markdown、raw 文件中写入 API key、token、secret-key URL、本地绝对路径或未脱敏私密数据。
